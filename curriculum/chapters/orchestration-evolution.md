# 5. 编排、适配与测试时演化

编排层决定：回合顺序、工具可见性、何时检索/重试/委派/停止。很多「模型不行」其实是编排合约过弱或过载。

## 三条演进路线

1. **手写更好的 harness**  
   固定生产循环 → 结构化合约（原生 tool call、compaction、零 token 等待……）

2. **自动适配 harness**  
   从故障轨迹发现指令/工具/编排改动；让 SLM 在重复业务流上逼近大模型性价比。

3. **测试时 / 在线演化**  
   不改权重、不要金标，只演化可执行控制程序（如 TTHE）；或权重–harness–解 共同演化（HASE）。

## 实务注意

- 高级编排功能有 **capability floor**：弱模型暴露 sub-agent / 复杂 MCP 可能净回退
- 应按模型档位降级暴露工具目录与委派
- 路由不仅看题面难度，还要看「将行使的编排特征」

### 关键要点

- 编排是可学习、可版本化的控制面
- 适配与演化把 harness 从静态基础设施变成优化对象
