# 2. 怎么用这套知识库

## 推荐阅读路径

1. **导论**（本章为止）——对齐术语
2. **成本 → 上下文 → 编排**——三条主线，对应最多 A 档实证
3. **合约 / 评估 / 安全**——生产化必补
4. **论文总览 + A 档精读**——按档深挖

有后端 / 客户端经验转 Agent：优先吃透「成本经济学」「上下文」「评估」三章——这往往是 demo 与生产的分界。

## 论文分档怎么读

| 档 | 含义 | 读法 |
|----|------|------|
| **A** | 主张清晰，有受控对照或评测审计 | 必读；有核心介绍长文 |
| **B** | 有实证但有软肋 | 按兴趣扫摘要与理由 |
| **C** | 愿景 / 基建 / 弱相关 | 当地图，不优先精读 |

评级基于**摘要级审阅**，不是全文同行评审。每条论文带 `indexed_at`（加入索引时间），便于追踪「哪天入库」。

## 内容如何更新

1. 往 `indexes/agent-harness/index.json` 加论文（或用 `_add_paper.py`）
2. 给论文打 `tags`（决定挂到哪一章）
3. A 档写 `papers/a/<id>.md`
4. `python scripts/build_site.py` 重建站点
5. 推送到 `main` → GitHub Actions 自动发布 Pages

标签与章节的映射写在 `curriculum/catalog.json`。若一篇论文应强制出现在某章，可在条目中加 `knowledge_ids: ["token-economics"]`。
