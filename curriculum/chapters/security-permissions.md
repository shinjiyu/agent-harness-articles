# 8. 安全、MCP 与权限

编码 / 工具代理在有限监督下读仓库、调工具、跑 shell。安全问题大量发生在**执行层与元数据层**，而不是聊天措辞。

## 关注面

| 面 | 典型问题 |
|----|----------|
| MCP 元数据 | 审批视图与注入模型的字节不一致（如不可见 Unicode TAG） |
| 规格不足 | 任务完成但越界猜动作 |
| 权限管理 | 用户参与轴：疲劳、过度拒绝、AI 辅助决策 |
| 执行隔离 | 沙箱、能力、TOCTOU、出口控制文献碎片化 |
| 双用途域 | 拒答校准（合法工作 vs 隐蔽风险） |

## 设计原则

- 审批所见 == 模型所得（保真）
- 失败要 typed，副作用要可丢弃
- 权限助手需情景敏感，没有万能设计
- 安全评测与完成率评测分开报

### 关键要点

- 执行安全需要专门系统化，不能只当「众多防御之一」
- 规格不足主要让代理「猜」，而不是「挂」
