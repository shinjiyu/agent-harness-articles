# Measuring Harness-Induced Belief Divergence in Multi-Step LLM Agents

- ArXiv: [2607.04528](http://arxiv.org/abs/2607.04528)
- Tier: **A**
- Indexed: 2026-07-13T17:32:31+08:00

## 核心介绍

软件代理基准通常只报告「是否完成任务」。本文指出：代理是通过 harness 到达结果的——harness 控制它看见什么、能做什么、修什么故障、验证什么状态、记录什么证据。即便**任务、环境与基座 LLM 固定**，不同 harness 仍会改变代理的**多步信念**，进而影响后续决策。

### 关键贡献

- 提出 **belief rollout 诊断**：在进展、风险、可恢复性、约束、故障模式、不确定性、未来成功、维修成本、替代工具下的下一步行动等维度，引出结构化 K 步轨迹。
- 定义 **跨 harness 信念分歧**，并分解为界面导致的「到达项」与地平线依赖的「增长项」。
- 引入免训练协议 **BIWM**：规范观察、记录审查分支、扩展修复跟踪、记录验证掩码、影子执行风险分支等，使信念轨迹可在不同 harness 视图间对齐比较。

### 关键洞察

阻止操作、压缩修复、选择性验证、成本意识证据修剪等，往往**仍能保住最终成功**，但会改写推动后续决策的信念。因此 harness 设计是评测中的**实验变量**，不是可忽略的实现细节。

### 为何进 A 档

把社区常忽略的评测混淆因素讲清楚，并给出可操作的诊断框架；对本索引后续比较不同 harness 论文结果具有方法论约束作用。

### 阅读时注意

当前证据主要来自受控编码任务与公共基准压力测试；外推到对话/具身等域需谨慎。代码见摘要中的 GitHub 链接。
