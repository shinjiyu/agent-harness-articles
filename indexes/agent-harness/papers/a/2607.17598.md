# Is Progressive Disclosure All You Need for Long-Context Agents?

- ArXiv: [2607.17598](http://arxiv.org/abs/2607.17598)
- Tier: **A**
- Indexed: 2026-07-21T17:20:00+08:00

## 核心介绍

长文档 QA 通常二选一：整本塞上下文，或外挂检索器。Agent Skills 的**渐进披露**（按查询逐级展开：短描述 → 具体段落）被实践者迅速采纳用于书级理解，但证据一直是轶事级。本文做了第一个受控研究：在 InfiniteBench 上，**3 个 agent harness × 3 个模型族**，比较裸文档导航、多种 Skills 包设计与经典混合检索器。

### 主要主张

- **单本书**：增益取决于 harness——代理裸导航差时增益大；强 harness 自己就会分块检索，增益趋近于零
- **多本书**：裸文档导航崩溃，一层渐进披露退化更慢并反超
- **两层路由从不加分**，有时直接掰断准确率——**一层就够**
- 一句话：渐进披露**买的是上下文，不是智能**；强代理自己能定位段落时它是冗余的，语料超出上下文时它是决定性的

### 为何进 A 档

对当下最流行的 Skills 实践给出首个受控证据，且结论以 harness 为条件变量——直接呼应本索引主线「能力应归因于模型–harness 配置」。对「要不要给代理配 Skills 包、配几层」是可执行结论。

### 阅读时注意

限 InfiniteBench 书级 QA 任务；「强 harness」的具体判据与 Skills 包构造方式影响外推，见原文设定。
