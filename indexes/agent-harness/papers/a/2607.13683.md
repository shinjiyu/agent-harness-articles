# Self-Evolving Agent Harnesses via Gated Semantic Quality-Diversity

- ArXiv: [2607.13683](http://arxiv.org/abs/2607.13683)
- Tier: **A**
- Indexed: 2026-07-16T17:54:00+08:00

## 核心介绍

冻结模型时，真正能动的杠杆往往是 **harness**（提示、注入知识、运行时控制、配置）。自演化的难点不在「生成补丁」，而在**哪些改动真正有效、且不是测量噪声或过拟合训练任务**。本文给出一套自演化 agent-harness 框架：语言模型负责诊断失败并提案，**采样、测量与显著性检验全部由确定性代码拥有**，使被记入账本的改进「构造上可信」。

### 机制要点

- **提案与计分分离**：LLM 只提案；信用归属不交给自生成反馈
- **GSME 归档**：按 (WHERE × WHY) 病理类别做门控质量–多样性归档，而不是按「修了哪些题」——作为抗过拟合归纳偏置
- **密封测试**：演化结束后才对 sealed test 打一次分，避免偷看测试调参

### 主要主张

七域、冻结开源权重模型上，train 选出的 harness 在 sealed test 上获得 **+9–15.5pp**，并保留训练增益的 **86–147%**（证据是泛化而非过拟合）。胜出补丁跟踪的是模型的**主导病理**，而非模型大小/家族；换模型可能换病理与补丁，但「诊断–记分」回路可迁移。

### 为何进 A 档

把自演化 harness 的可信性问题落到可审计的工程分工（谁提案、谁计分、如何防过拟合），并有密封测试数字支撑。是编排/自演化知识点的核心新文，可与 TTHE、Better Harnesses Smaller Models 对照读。

### 阅读时注意

增益依赖病理分类粒度、门控阈值与域选择；「保留 >100% 训练增益」需结合方差与基线定义细读。完整设定见 PDF。
