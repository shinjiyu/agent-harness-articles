# Automated Discovery Has No Universally Superior Harness

- ArXiv: [2607.18235](http://arxiv.org/abs/2607.18235)
- Tier: **A**
- Indexed: 2026-07-21T17:20:00+08:00

## 核心介绍

OpenEvolve、TTT-Discover 这类自动发现系统常被当作**通用 harness**使用，但它们其实是把归档、父代选择、探索、预算分配等多个设计选择捆成一个配方的复合系统。发现类任务跑一次很贵且天然随机，现有比较往往试验次数太少，分不清「方法改进」与「run-to-run 方差」。本文把这两类 harness **系统分解成组件**，在预算对齐下评测 **30 个 harness × 12 个模型-问题对**，用超过 **310 万次 LLM rollout** 做重复试验统计。

### 主要主张

- **无普适最优**：没有任何固定 harness 在所有模型-问题对上可靠占优；OpenEvolve 系变体普遍输给更简单的替代
- **harness 应视为超参数**而非通用配方，要按问题和底座模型调
- **早期进展可预测终局**：据此设计预算对齐的自适应分配——起多套 harness、剪弱留强、算力向幸存者集中，胜过随机固定选择

### 为何进 A 档

用重复试验方差控制直接打「刷一次榜就下结论」的比较范式，与 Harness-Bench（2605.27922）「能力=模型–harness 配置」一脉相承，并给出可操作的自适应分配方案。对「要不要换 harness/怎么选」这类工程决策有直接指导。

### 阅读时注意

结论限于发现/进化搜索类任务（非通用代理任务）；「早期进展预测终局」的适用边界与剪枝误杀率需读原文。
