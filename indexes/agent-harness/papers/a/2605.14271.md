# Auditing Agent Harness Safety

- ArXiv: [2605.14271](http://arxiv.org/abs/2605.14271)
- Tier: **A**
- Indexed: 2026-07-21T17:20:00+08:00

## 核心介绍

harness 可以在**轨迹中途**访问未授权资源、把上下文泄露给错误的代理，却仍返回一个正确无害的最终答案——而多数安全基准只给终态打分，根本看不见这类失败。本文提出 **HarnessAudit**：对完整执行轨迹做审计，覆盖**边界合规、执行保真、系统稳定**三个维度，重点针对风险最集中的多代理 harness。配套 **HarnessAudit-Bench**：210 个任务 × 8 个真实域，单代理与多代理双配置，内嵌安全约束。

### 主要发现（10 个 harness 配置 × 前沿模型 × 3 个多代理框架）

- **任务完成与安全执行错位**：完成率高不代表执行安全，违规随轨迹长度**累积**
- 安全风险随域、任务类型、代理角色显著变化
- 违规集中在**资源访问**与**代理间信息传递**两处
- 多代理协作**扩大**攻击面

### 为何进 A 档

把 harness 安全从「输出审查」推进到「轨迹审计」，与本索引 Janus（权限管理）、Harness-MU（多用户治理）、Safety Sentry(三路路由)构成安全知识点的评测基建核心。「输出正确 ≠ 执行安全」对企业部署是直接可用的验收标准。

### 阅读时注意

基准域覆盖与嵌入约束的代表性决定外推；三维审计指标的权重与判定阈值见原文协议。
