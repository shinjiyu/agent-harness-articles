# Do Agent Optimizers Compound? A Continual-Learning Evaluation on Terminal-Bench 2.0

- ArXiv: [2607.14004](http://arxiv.org/abs/2607.14004)
- Tier: **A**
- Indexed: 2026-07-16T17:54:00+08:00

## 核心介绍

多数 agent 优化论文只报**单次、静态**基准增益，却不测部署里真正要紧的设定：失败与新任务不断到来时，优化能否**递归施加且增益是否复利**。本文用 Terminal-Bench 2.0 难题构造两阶段 continual-learning 评测，在相同优化预算下比较三种 harness 优化路线：GEPA、Meta Harness、RELAI-VCL（Verifiable Continual Learning）。

### 关键设计

- Phase 1：对固定难任务集优化，对应「常规单阶段报数」
- Phase 2：引入新任务后再给一轮优化预算，检验**迁移**与**继续提升**
- 三方法共享预算与基线 agent，避免「各自刷榜」不可比

### 主要主张

静态单阶段下三者都高于基线；一旦进入新任务流，分化剧烈：

- **GEPA**：迁移后甚至低于未优化基线（捷径解）
- **Meta Harness**：迁移尚可，但第二轮预算几乎不再涨
- **RELAI-VCL**：唯一同时做到正向迁移 + 第二轮继续提升，终身平均 pass 最高（76.4% vs GEPA 66.0 / Meta 64.6 / 基线 58.7）

关键观察：只有把**回归控制**写进优化环，增益才会复利——否则优化器会找到过拟合当前失败的捷径。

### 为何进 A 档

把「harness 自优化」从一锤子刷分推进到**可复利的 continual 设定**，并给出可操作的工程归纳：回归控制是抗捷径的偏置。直接指导是否采用某类 harness optimizer。

### 阅读时注意

结论绑定 Terminal-Bench 2.0 难任务子集与所选三种优化器实现；外推到其他基准或优化器族时需复核。完整协议与曲线见 PDF。
