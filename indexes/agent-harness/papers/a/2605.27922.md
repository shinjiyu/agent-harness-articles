# Harness-Bench: Measuring Harness Effects across Models in Realistic Agent Workflows

- ArXiv: [2605.27922](http://arxiv.org/abs/2605.27922)
- Tier: **A**
- Indexed: 2026-07-14T17:31:00+08:00

## 核心介绍

现有代理基准往往抽象掉执行层、比较整套 agent 系统，或把 harness **固定不变**——导致很难单独研究「编排/执行层」带来的差异。Harness-Bench 把 **harness 配置**当作实验变量：在共享任务环境、预算与评测协议下，跨多个模型后端比较代表性 harness 配置，并保留各自原生执行行为。

### 关键设计

- **106** 个沙箱离线任务，来自实用 agent 使用模式，人工审 realism / 可解 / oracle 可检查 / 完整性
- 每次运行记录：最终产物、执行轨迹、用量统计、验证器输出（不止 completion）
- **5,194** 条执行轨迹

### 主要主张

跨 model–harness 配对，在完成率、过程质量、效率与失败行为上差异显著。因此代理能力应报告为 **模型–harness 配置**，而不是只归因于基座模型。分析还标出一类复发的 **execution-alignment** 失败：推理看起来合理，却与工具反馈、工作区状态、证据或可验证输出合约脱节。

### 为何进 A 档

与 [The Harness Effect](2607.06906.md) 互补：后者锁任务换编排看**成本经济学**；本文在更广配置矩阵上量化 **harness 对能力/效率/失败模式** 的影响，并给出可复现诊断基准。是本索引「评估与评测偏差」知识点的核心文献。

### 阅读时注意

诊断基准依赖所选 harness 代表集与任务构造；外推到未覆盖的生产框架时需谨慎。完整数字与协议见 PDF。
