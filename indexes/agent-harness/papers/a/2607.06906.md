# The Harness Effect: How Orchestration Design Sets the Token Economics of Enterprise Agentic AI

- ArXiv: [2607.06906](http://arxiv.org/abs/2607.06906)
- Tier: **A**
- Indexed: 2026-07-13T17:32:31+08:00

## 核心介绍

企业代理开发常陷入「用 token 换能力」：更长推理、更多回合、更大工具载荷与重放上下文，导致**单任务成本上涨快于任务价值**。本文把对抗这种 token 最大化的关键杠杆定位为 **harness（编排层）**——组装上下文、暴露工具、排序回合、委派工作，并承载可观测性与治理。

### 关键设计

在 **22 个锁定评估任务**、**6 个基础模型**上做受控交换：只改编排层（冻结的传统生产循环 vs Writer Agent Harness），模型不变。从而把「换模型」与「换 harness」的效果隔离开。

### 主要结果（摘要口径）

- 混合成本约 **−41%**（$0.21 → $0.12）
- 中位墙钟时间约 **−44%**（48s → 27s）
- 每任务 token 约 **−38%**（14.2k → 8.8k）
- 任务质量大致持平或略升（0.78 → 0.81，样本下偏方向性）
- 效率增益几乎 **模型无关**；质量增益与模型基线强度高度相关（文中称 *harness leverage*）

### 为何进 A 档

实验变量控制干净（锁任务、锁模型、只换 harness），并直接回答工程问题：**编排层对成本的影响可以大过换模型菜单本身**。是理解「agent harness」经济意义时的首选实证文。

### 阅读时注意

企业任务代表性与外部可复现性需在全文核对；质量结论在该样本量下偏方向性，不宜过度外推到开放域。
