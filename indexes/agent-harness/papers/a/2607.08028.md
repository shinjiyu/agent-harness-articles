# From Prompts to Contracts: Harness Engineering for Auditable Enterprise LLM Agents

- ArXiv: [2607.08028](http://arxiv.org/abs/2607.08028)
- Tier: **A**
- Indexed: 2026-07-13T17:32:31+08:00

## 核心介绍

企业 LLM 应用常从「提示 + 检索上下文」原型起步；产品化则要求源边界、实体路由、应答合同与可复现跟踪。本文提出一种 **harness 工程模式**：把确定性行为下沉到可替换的组合边界上的**代码、清单、schema 与验证工件**，同时让**有源声明**仍是运行时答案的权威。

### 评估设置（摘要口径）

在五个韩国企业集团（25 家上市公司）的公共数据切片上实例化，围绕三个研究问题：

1. **合约保留**：源接地、实体路由、跟踪、输出卫生、推荐语言等合约在固定验证场景中得以保持；故障注入可被验证者标记。
2. **模型替换**：三个托管模型、270 次组合边界运行均通过检查；失败限于模型侧并被记录。
3. **代码层 vs 纯提示**：仅改执行层时，纯提示无法阻止推荐语言/内部跟踪泄漏；harness 可完全阻断。外挂护栏也能挡，但过度拒绝（效用 88/120），而 harness 保留满效用（120/120）。

### 为何进 A 档

把「可审计企业代理」从口号落成可复用工程模式，并有合约、模型替换与消融三类证据；对从原型走向治理就绪系统有直接参考价值。

### 阅读时注意

领域为特定企业公开数据切片；跨行业、跨语言外推需另验。重点应学其**边界与工件分层**，而非照搬业务规则。
